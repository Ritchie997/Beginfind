# BeginFind Server

Серверная часть приложения BeginFind с поддержкой базы данных для мессенджера и ibripedia.

## Структура проекта

- `index.js` - основной серверный файл
- `messenger.db` - база данных мессенджера (создается автоматически)
- `ibripedia.db` - база данных ibripedia (создается автоматически)
- `public/` - статические файлы (веб-интерфейс)

## Установка зависимостей

```bash
npm install
```

## Переменные окружения

Перед запуском сервера создайте файл `.env` в корне проекта на основе `.env.example`:

```bash
cp .env.example .env
```

Доступные переменные:
- `PORT` - Порт для сервера (по умолчанию: 3002)
- `HOST` - Хост для сервера (по умолчанию: localhost)
  - Для локальной разработки: localhost
  - Для доступа из локальной сети: 0.0.0.0

## Запуск сервера

### В режиме разработки
```bash
npm run dev
```

### В продакшен режиме
```bash
npm start
```

Сервер будет доступен по адресу: http://localhost:3002

Для доступа из локальной сети установите HOST=0.0.0.0 в файле .env

## API Endpoints

### Мессенджер

- `GET /api/messages` - Получить все сообщения
- `POST /api/messages` - Отправить новое сообщение

### Ibripedia (википедия)

- `GET /api/articles` - Получить список всех статей
- `GET /api/articles/:id` - Получить статью по ID
- `POST /api/articles` - Создать новую статью
- `PUT /api/articles/:id` - Обновить статью
- `DELETE /api/articles/:id` - Удалить статью

## Базы данных

Сервер использует SQLite для хранения данных:

1. **messenger.db** - содержит данные мессенджера (сообщения)
2. **users.db** - содержит данные пользователей и ролей
3. **articles.db** - содержит данные статей и категорий
4. **servers.db** - содержит данные серверов, ролей на серверах и участников

## Веб-интерфейс

Сервер также обслуживает статические файлы из папки `public/`, включая тестовый интерфейс для проверки API.

## Разработка

Для разработки используется:
- Node.js
- Express.js
- SQLite3
- Nodemon (для автоматической перезагрузки при изменениях)
