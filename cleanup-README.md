# Система очистки мусорных файлов

## Описание

В проекте реализована система автоматической очистки мусорных файлов, которые остаются на сервере после удаления статей или сообщений. Это позволяет экономить дисковое пространство сервера.

## Как работает

Система очистки работает следующим образом:

1. Сканирует все файлы в директории `public/uploads`
2. Получает список всех изображений, используемых в статьях (из поля `image` таблицы `articles`)
3. Получает список всех изображений, используемых в сообщениях (из поля `content` таблицы `messages`)
4. Сравнивает файлы в директории с используемыми файлами
5. Удаляет файлы, которые не используются

## Автоматическая очистка

Автоматическая очистка запускается:
- При старте сервера
- Ежедневно в 3:00 по московскому времени

## Ручная очистка

Для ручного запуска очистки можно использовать команду:

```bash
npm run cleanup
```

## Файлы системы

- `cleanup.js` - основной скрипт очистки мусорных файлов
- `scheduled-cleanup.js` - скрипт для автоматического запуска очистки по расписанию

## Технические детали

Алгоритм очистки учитывает:
- Изображения, прикрепленные к статьям в поле `image`
- Изображения, встроенные в текст сообщений
- Все файлы в директории `public/uploads`
